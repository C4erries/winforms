# Учет товаров магазина канцтоваров

Учебный проект на C# Windows Forms с PostgreSQL. Приложение ведет учет товаров магазина канцтоваров по приходным и расходным накладным, хранит платежи поставщикам и строит отчеты по движению товаров и задолженности.

## Как запустить

1. Поднять PostgreSQL:

```powershell
make db
```

2. Заполнить базу тестовыми данными:

```powershell
make seed
```

3. Запустить приложение:

```powershell
make run
```

Также можно открыть `StationeryStore.sln` в Visual Studio и запустить проект `StationeryStore`.

По умолчанию приложение подключается к базе:

```text
Host=localhost;Port=5432;Username=postgres;Password=12345;Database=stationery
```

Если нужно, строку подключения можно переопределить через переменную окружения `POSTGRES_CONNECTION_STRING`.

## Структура проекта

`StationeryStore.sln` - обычный solution-файл для Visual Studio.

`StationeryStore.slnx` - новый формат solution, созданный dotnet SDK. Основной для сдачи лучше использовать `StationeryStore.sln`.

`docker-compose.yml` - PostgreSQL в Docker. При первом старте контейнер применяет `db/init.sql`.

`Makefile` - удобные команды для запуска БД, приложения, сидов и очистки.

`db/init.sql` - создание таблиц, связей, ограничений и триггеров пересчета суммы накладной.

`db/seed.sql` - демонстрационные данные: группы товаров, товары, поставщики, накладные, состав накладных и платежи.

`src/StationeryStore/Program.cs` - точка входа приложения. Запускает главную форму.

`src/StationeryStore/Data/Db.cs` - простой класс для подключения к PostgreSQL и выполнения SQL-запросов.

`src/StationeryStore/Forms/MainForm.cs` - главная форма приложения. В ней создаются вкладки справочников, накладных, платежей и отчетов.

`src/StationeryStore/Forms/CrudTabPage.cs` - универсальная вкладка CRUD для простых таблиц: группы товаров, товары, поставщики, платежи.

`src/StationeryStore/Forms/InvoiceTabPage.cs` - вкладка накладных. Сверху список накладных, снизу состав выбранной накладной. Это выполняет требование про две связанные главные таблицы на одной форме.

`src/StationeryStore/Forms/ReportsTabPage.cs` - вкладка отчетов. Здесь строится движение товаров за период, экспорт в Excel и отчет задолженности по дням с круговой диаграммой.

`src/StationeryStore/Forms/EditRecordForm.cs` - универсальное окно добавления/изменения записи.

`src/StationeryStore/Forms/UiStyles.cs` - общий внешний вид таблиц, кнопок и секций.

`src/StationeryStore/Services/ExportService.cs` - экспорт отчета в Excel и накладной в Word.

`example/` - исходный пример проекта, на основе которого делалась структура.

## Как устроен интерфейс

Приложение сделано через вкладки:

- `Группы товаров` - справочник групп, например бумага, папки, настольные принадлежности.
- `Товары` - справочник товаров. Каждый товар относится к группе.
- `Поставщики` - справочник поставщиков.
- `Накладные` - приходные и расходные накладные. В верхней таблице накладные, в нижней таблице товары выбранной накладной.
- `Платежи` - оплаты по приходным накладным. Одна накладная может оплачиваться несколькими платежами.
- `Отчеты` - движение товаров за период, экспорт в Excel, задолженность перед поставщиками по дням и диаграмма.

Добавление, изменение и удаление выполняются с выделенной строкой. При изменении открывается форма с уже существующими значениями, то есть запись изменяется, а не вводится заново.

## Бизнес-семантика

Предметная область - магазин канцтоваров.

Магазин получает товар от поставщиков по приходным накладным. В каждой приходной накладной есть список товаров, количество и цена. На основании этих данных увеличиваются остатки товаров.

Расходные накладные отражают выбытие товара из магазина. В них также указываются товары, количество и цена. На основании этих данных уменьшаются остатки товаров.

Поставщик может получить оплату не сразу всей суммой, а несколькими платежами. Поэтому платежи вынесены в отдельную таблицу и связаны с накладной. Задолженность считается как сумма приходных накладных минус сумма платежей.

Отчет движения товаров за период показывает:

- остаток товара на начало периода;
- приход за период;
- расход за период;
- остаток на конец периода.

Отчет строится только по выбранным группам товаров. Это соответствует требованию про таблицу движения товаров для товаров из указанных групп.

Отчет задолженности по дням показывает, как менялась общая задолженность перед поставщиками в течение выбранного периода. По этим данным строится круговая диаграмма.

## Схема базы данных

В базе больше четырех таблиц, потому что предметная область требует справочники, документы, состав документов и платежи.

`product_group` - группы товаров. Нужны для классификации товаров и фильтрации отчета движения.

`product` - товары магазина. Каждый товар связан с группой через `group_id`. В интерфейсе показывается название группы, а не ID.

`supplier` - поставщики. Используются в накладных и позволяют считать задолженность перед поставщиками.

`invoice` - накладные. В этой таблице хранится шапка документа: поставщик, номер, дата, тип накладной, примечание и итоговая сумма. Тип бывает `income` для приходной и `expense` для расходной.

`invoice_item` - состав накладной. Здесь хранится список товаров внутри конкретной накладной: товар, количество и цена. Эта таблица связывает накладные и товары.

`payment` - платежи по накладным. Одна приходная накладная может иметь несколько платежей, поэтому платежи не хранятся прямо в `invoice`.

Связи сделаны внешними ключами:

- товар ссылается на группу товаров;
- накладная ссылается на поставщика;
- позиция накладной ссылается на накладную и товар;
- платеж ссылается на накладную.

Для `invoice_item` есть триггеры. Они автоматически пересчитывают `invoice.total_sum` при добавлении, изменении или удалении товара в накладной. Это нужно, чтобы итоговая сумма накладной всегда соответствовала ее составу.

## Что важно сказать на защите

Проект реализует CRUD для всех основных таблиц. Для связанных данных используются выпадающие списки с названиями, а не ручной ввод ID. В накладных две связанные таблицы находятся на одной форме: список накладных и состав выбранной накладной. Есть экспорт накладной в Word и экспорт отчета движения товаров в Excel. База данных PostgreSQL запускается через Docker, схема создается SQL-скриптом.
